## AI-Code-Generator

AI-Code-Generator is a simple FastAPI-based service that loads source code from a public GitHub repository, builds embeddings and a FAISS index over the files, and lets you ask questions about the code using a local LLM (via Ollama).

### Features
- Download a GitHub repository as a ZIP and extract text/code files
- Create sentence-transformer embeddings for repository files
- Build a FAISS vector index for semantic search over the codebase
- Ask natural-language questions about the codebase with context selected via vector search
- Get explanations generated by an Ollama-hosted `llama3` model

### Requirements
- Python 3.9+
- Ollama installed and running locally (with the `llama3` model pulled)
- Internet access to download GitHub repositories

### Installation
1. Create and activate a virtual environment (optional but recommended).
2. Install dependencies:
	 ```bash
	 pip install fastapi uvicorn requests ollama sentence-transformers faiss-cpu numpy
	 ```

### Running the API
Start the FastAPI app with Uvicorn from the project root:

```bash
uvicorn app:app --reload
```

By default this runs on `http://127.0.0.1:8000`.

### API Endpoints

#### `POST /load_repo`
- **Body (JSON or form field):** `repo_url` – full GitHub repo URL, e.g. `https://github.com/owner/repo`
- **Behavior:**
	- Downloads the repository ZIP (tries `main` then `master` branch)
	- Extracts a subset of text/code files
	- Stores file contents in memory
- **Response:**
	- `{ "files_loaded": <int> }`

#### `POST /ask`
- **Body (JSON or form field):** `question` – natural-language question about the loaded repository
- **Behavior:**
	- Embeds the question using the same sentence-transformer model
	- Uses FAISS to retrieve the top-k most relevant files
	- Concatenates trimmed content from those files as context
	- Sends a prompt to Ollama’s `llama3` model
	- Returns the model’s answer

### Notes & Limitations
- Currently only one repository is held in memory at a time (global state).
- Context uses only a subset of files and characters, so very large repos may be summarized only partially.
- Error handling is minimal; make sure the repository is public and reachable.

### Future Improvements
- Add authentication and better error reporting.
- Support multiple concurrent repositories and sessions.
